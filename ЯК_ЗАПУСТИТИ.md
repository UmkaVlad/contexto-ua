# Як запустити гру Contexto UA

Є **два способи** грати: **без сервера** (демо) або **з сервером** (повна версія з усіма словами).

---

## Спосіб 1. Грати без сервера (демо)

Якщо сервер не вдається запустити, можна грати одразу:

1. Відкрийте у браузері файл:
   ```
   game_files/frontend/index.html
   ```
   (Подвійний клік по файлу або перетягніть його у вікно браузера.)

2. Гра запуститься в **демо-режимі**: обмежений список українських слів, але інтерфейс і правила ті самі. Позиції рахуються приблизно.

Щоб потім перейти на повну версію — встановіть Python і запустіть сервер (Спосіб 2).

---

## Спосіб 2. Запустити сервер (повна версія)

Помилка **«Немає зв'язку із сайтом»** означає, що сервер ще не запущений. Помилки **«No such file or directory: .venv/bin/pip»** означають, що на Mac не вдалося створити віртуальне середовище (часто через відсутність Xcode). Зробіть так:

## Крок 1. Відкрийте Термінал

- **macOS:** Програми → Утиліти → Термінал (або пошук «Terminal»).
- Або в Cursor: меню **Terminal → New Terminal**.

## Крок 2. Перейдіть у папку гри

```bash
cd "/Users/Umka/Documents/Vlad Contexto Game/game_files"
```

(Або перетягніть папку `game_files` у вікно Терміналу — шлях підставиться сам.)

## Крок 3. Запустіть сервер

**Варіант A — скрипт (рекомендовано):**

```bash
chmod +x run.sh
./run.sh
```

**Варіант B — вручну:**

```bash
python3 -m venv .venv
source .venv/bin/activate
pip install -r backend/requirements.txt
uvicorn backend.contexto_backend:app --host 0.0.0.0 --port 8000
```

## Крок 4. Відкрийте гру в браузері

Коли в Терміналі з’явиться повідомлення, що сервер запущено:

1. Відкрийте браузер.
2. У рядку адреси введіть **одну** з цих адрес:
   - **http://localhost:8000/**
   - **http://127.0.0.1:8000/**

**Не вводьте** в браузері `0.0.0.0` — це адреса для сервера, браузер до неї не підключиться.

Гра має відкритися. Не закривайте вікно Терміналу — поки сервер працює, сайт буде доступний.

Зупинити сервер: у Терміналі натисніть **Ctrl+C**.

---

## Якщо щось не встановлюється

- **«No developer tools were found» / «.venv/bin/pip: No such file or directory»** — системний Python на Mac потребує Xcode. Краще встановити Python окремо:
  1. Відкрийте [python.org/downloads](https://www.python.org/downloads/)
  2. Завантажте **macOS 64-bit installer** (Python 3.12 або новіший)
  3. Встановіть, перезапустіть Термінал і виконайте знову: `./run.sh`
- **«python3: command not found»** — встановіть Python з [python.org](https://www.python.org/downloads/) (версія 3.10 або новіша).
- **Помилки при `pip install`** — спробуйте: `pip install --upgrade pip`, потім знову `pip install -r backend/requirements.txt`.
- **Порт 8000 зайнятий** — запустіть на іншому порту:
  ```bash
  uvicorn backend.contexto_backend:app --host 0.0.0.0 --port 8080
  ```
  і відкрийте в браузері: **http://localhost:8080/**

Якщо сервер запустити не вдається — використовуйте **Спосіб 1** (відкрити `frontend/index.html` у браузері, грати в демо-режимі).
